Use these standards when creating brand surfaces, documentation pages, generated redirect pages, and operator-facing interfaces.

The design language is deliberately practical. It should help contributors make consistent choices without turning every small change into a design review.

The current system is defined by the Hugo layouts, Tailwind tokens, PostCSS pipeline, self-hosted fonts, Mermaid support, Chroma code highlighting, and the documentation rhythm in `assets/css/main.css`.

The foundation pages adapt useful patterns from Red Hat's public design-system guidance for [color](https://ux.redhat.com/foundations/color/), [grid](https://ux.redhat.com/foundations/grid/), [iconography](https://ux.redhat.com/foundations/iconography/), [links](https://ux.redhat.com/foundations/interactions/links/), [spacing](https://ux.redhat.com/foundations/spacing/), and [typography](https://ux.redhat.com/foundations/typography/font-family/) while keeping vanityURLs-specific tokens, assets, and product needs.

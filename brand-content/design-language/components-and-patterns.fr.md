Le systeme de marque doit reutiliser les composants existants avant d'en creer de nouveaux. Les motifs partages rendent les pages previsibles et reduisent la derive CSS.

## Motifs de documentation

Utilisez les shortcodes Hugo et composants CSS existants pour :

- Encadres et avis.
- Etapes et parcours de configuration.
- Arborescences de fichiers et structure de depot.
- Blocs de code avec bouton de copie.
- Tableaux pour jetons, actifs et notes de compatibilite.
- Sections details pour les explications secondaires.
- Cartes pour les listes de navigation ou les telechargements.

## Motifs produit

Les surfaces de redirection generees et les interfaces operateur doivent etre reliees a la documentation sans devenir des pages de documentation.

- Utilisez des en-tetes compacts et des libelles d'etat clairs.
- Rendez les actions principales distinctes.
- Preferez tableaux, listes de definitions et panneaux courts pour les donnees operationnelles.
- Affichez les valeurs techniques completes quand la verification compte.
- Evitez les cartes decoratives imbriquees.

## Decisions de motif

Creez un nouveau composant seulement si un motif existant ne peut pas exprimer le contenu clairement. Si un composant apparait sur plus d'une page, documentez ici son usage et ses contraintes avant qu'il se propage.

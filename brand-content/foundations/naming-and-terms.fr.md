Utilisez des noms coherents pour distinguer le produit, le site public, le site de marque et les surfaces de redirection generees.

## Projet et domaines

| Nom                     | Utilisation                                               |
| ----------------------- | --------------------------------------------------------- |
| `vanityURLs`            | Nom du projet et du produit                               |
| `vanityURLs.link`       | Domaine public de documentation et de produit             |
| `brand.vanityURLs.link` | Domaine des standards de marque et du systeme de design   |
| `v8s.link`              | Instance de demo, exemples de badges et exemples compacts |

Gardez le style `vanityURLs` dans le texte. Dans les URLs et noms d'hote, utilisez la casse reelle seulement quand elle aide la reconnaissance; DNS n'est pas sensible a la casse.

## Termes produit

| Terme        | Sens                                                           |
| ------------ | -------------------------------------------------------------- |
| Redirection  | Requete de lien court resolue vers une cible configuree        |
| Alias        | Chemin court ou mot-cle utilise pour atteindre une cible       |
| Instance     | Redirecteur deploye avec les valeurs par defaut ou surchargees |
| Page generee | Page produite par le redirecteur pour l'expansion ou le repli  |
| Badge        | Lien visuel compact qui annonce ou previsualise un alias       |
| Operateur    | Personne qui configure, deploie ou audite une instance         |

## Regles d'ecriture

- Preferez les termes produit aux synonymes inventes.
- Definissez un terme a sa premiere apparition dans une page pour debutants.
- Utilisez le format code pour les chemins, hotes, commandes, variables d'environnement, jetons et fichiers litteraux.
- Gardez les affirmations legales ou de marque factuelles et minimales jusqu'a l'existence d'une politique formelle.

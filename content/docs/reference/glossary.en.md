This glossary defines the terms used across vanityURLs documentation. Use it when a page mentions a concept before explaining the file or workflow behind it.

<dl>
  <dt><dfn>Access application</dfn></dt>
  <dd>A Cloudflare Zero Trust application that protects selected paths, such as <code>/_stats/*</code> and <code>/_tests/*</code>, before the Worker serves those pages.</dd>

  <dt><dfn>Access policy</dfn></dt>
  <dd>A Cloudflare Zero Trust rule that decides which identities can reach a protected Access application.</dd>

  <dt><dfn>Block policy</dfn></dt>
  <dd>The instance-owned rules in <code>custom/v8s-policies.json</code> that block unsafe domains, keywords, scanner paths, or unwanted traffic before redirect lookup.</dd>

  <dt><dfn>Build artifact</dfn></dt>
  <dd>A generated file under <code>build/</code> or <code>src/</code>. Edit source files in <code>custom/</code> or <code>defaults/</code>, then rebuild instead of editing artifacts directly.</dd>

  <dt><dfn>Custom overlay</dfn></dt>
  <dd>The instance-owned layer under <code>custom/</code>. It overrides or extends product defaults without editing upstream files.</dd>

  <dt><dfn>Default target</dfn></dt>
  <dd>The fallback destination used by a scheduled link when no time-based rule is currently active.</dd>

  <dt><dfn>Destination</dfn></dt>
  <dd>The long URL a short link redirects to. In the link format, it is stored in the <code>target</code> column.</dd>

  <dt><dfn>Generated registry</dfn></dt>
  <dd>The runtime redirect registry written to <code>build/v8s.json</code> during build from links, schedules, policies, and configuration.</dd>

  <dt><dfn>Identity provider</dfn></dt>
  <dd>A login source used by Cloudflare Access, such as GitHub, Google, or one-time PIN, to confirm who is trying to reach a protected page.</dd>

  <dt id="idempotent"><dfn>Idempotent</dfn></dt>
  <dd>A command or operation that can be run repeatedly with the same inputs without creating duplicate state or requiring a fresh setup. In vanityURLs, <code>npm run setup</code> rereads existing answers and updates the same generated files.</dd>

  <dt><dfn>Link registry</dfn></dt>
  <dd>The set of short-link records used by vanityURLs. Humans edit <code>custom/v8s-links.txt</code>; the build produces the generated registry consumed by the Worker.</dd>

  <dt><dfn>Operator</dfn></dt>
  <dd>The person, team, or organization responsible for the vanityURLs instance, public pages, trust contacts, and legal-page configuration.</dd>

  <dt><dfn>Owner label</dfn></dt>
  <dd>A short internal value in <code>v8s-links.txt</code> that identifies the person or team accountable for a link.</dd>

  <dt><dfn>Random slug</dfn></dt>
  <dd>A slug generated by <code>lnk</code> when you provide a destination but do not choose the short path yourself.</dd>

  <dt><dfn>Route</dfn></dt>
  <dd>The Cloudflare Workers mapping that sends requests for your short domain to the vanityURLs Worker.</dd>

  <dt><dfn>Runtime</dfn></dt>
  <dd>The Cloudflare Worker and static assets that serve redirects and pages after build and deployment.</dd>

  <dt><dfn>Scheduled link</dfn></dt>
  <dd>A short link whose destination can change by day, time window, and timezone using <code>custom/v8s-schedules.json</code>.</dd>

  <dt><dfn>Short domain</dfn></dt>
  <dd>The domain used for redirects, such as <code>v8s.link</code> or <code>go.example.com</code>.</dd>

  <dt><dfn>Slug</dfn></dt>
  <dd>The path after your short domain. In <code>https://v8s.link/docs</code>, the slug is <code>docs</code>.</dd>

  <dt><dfn>Splat link</dfn></dt>
  <dd>A link stored as <code>slug/*</code> that forwards the remaining path into a destination containing <code>:splat</code>.</dd>

  <dt><dfn>Tag</dfn></dt>
  <dd>A comma-separated label on a link row used to group links, describe purpose, or select tag-specific random slug lengths.</dd>

  <dt><dfn>Worker</dfn></dt>
  <dd>The Cloudflare Workers application that runs the vanityURLs redirector.</dd>
</dl>
